## üìû MAINTENANCE & UPDATES

### Updating Translations When Plugin Changes

```bash
# 1. Update POT template
wp i18n make-pot . languages/designsetgo.pot --domain=designsetgo

# 2. Merge into existing translations (preserves existing work)
msgmerge --update languages/designsetgo-es_ES.po languages/designsetgo.pot
msgmerge --update languages/designsetgo-fr_FR.po languages/designsetgo.pot
# Repeat for all languages...

# 3. Translate new strings
# (Use Poedit or text editor to add translations for new msgid entries)

# 4. Recompile MO files
msgfmt -o languages/designsetgo-es_ES.mo languages/designsetgo-es_ES.po
msgfmt -o languages/designsetgo-fr_FR.mo languages/designsetgo-fr_FR.po
# Repeat for all languages...
```

### Automated Update Script

```bash
#!/bin/bash
# update-translations.sh

cd languages

# Update POT
wp i18n make-pot .. designsetgo.pot --domain=designsetgo

# Merge and recompile all languages
for lang in es_ES fr_FR de_DE it_IT pt_BR ja zh_CN nl_NL ru_RU; do
    echo "Updating $lang..."
    msgmerge --update "designsetgo-$lang.po" designsetgo.pot
    msgfmt -o "designsetgo-$lang.mo" "designsetgo-$lang.po"
done

echo "‚úÖ All translations updated!"
```

### Adding New Strings

When adding new translatable strings to the plugin:

1. **Use `__()` or `_e()` functions:**
   ```php
   __('My new string', 'designsetgo')
   _e('Output directly', 'designsetgo')
   ```

2. **For JavaScript:**
   ```javascript
   import { __ } from '@wordpress/i18n';
   __('My new string', 'designsetgo')
   ```

3. **Update POT template:**
   ```bash
   wp i18n make-pot . languages/designsetgo.pot
   ```

4. **Merge into language files** (see above)

5. **Send new strings to translators** or use translation service
