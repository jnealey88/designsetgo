# Blobs Block - User Guide

**Version**: 1.0.0
**Category**: Design
**Keywords**: blob, shape, organic, morph, animate, gradient

## Overview

The **Blobs Block** creates organic, fluid blob shapes with smooth animations. Customize with gradients, images, and overlays. Perfect for modern hero sections, backgrounds, decorative elements, and eye-catching content containers.

**Key Features:**
- 6 unique blob shapes
- 5 animation options (morph, float, pulse, spin)
- Background image/gradient support
- Overlay effects with opacity control
- Adjustable size and animation speed
- Supports inner blocks for content

1. Insert the **Blobs** block (found in Design category).

4. Optionally add animation for movement.
5. Add content inside using inner blocks (headings, text, buttons).

## ‚öôÔ∏è Settings & Configuration

### Blob Shape Panel

**Shape Options:**
- **Shape 1-6**: Six pre-designed organic blob shapes

**Size**: Set blob dimensions (px, %, vw, vh)
- Default: 300px
- Suggested: 200px-600px for most uses

### Animation Panel

**Animation Type:**
- **Morph 1**: Smooth shape-shifting between variations
- **Morph 2**: Alternative morphing pattern
- **Float**: Gentle up/down floating motion
- **Pulse**: Scale in/out pulsing effect
- **Spin**: Continuous rotation
- **None**: No animation (static)

**Animation Duration**: Speed of animation (1s-20s)
- Fast: 2-4s
- Medium: 6-10s (recommended)
- Slow: 12-20s

**Animation Easing**: Motion curve (linear, ease, ease-in, ease-out, ease-in-out)

### Background
- **Color/Gradient**: Via Block Supports color panel
- **Background Image**: Upload image that fills blob shape
- **Background Size**: Cover, contain, etc.
- **Background Position**: Adjust image focal point

### Overlay Panel
- **Enable Overlay**: Toggle overlay layer
- **Overlay Color**: Color over background/image
- **Overlay Opacity**: 0-100% transparency

### Appearance
- **Alignment**: Left, Center, Right, Wide, Full
- **Spacing**: Margin and padding via Block Supports

## üí° Common Use Cases

### 1. Hero Section Background
Use **Shape 3** at 600px, gradient background, **Float** animation. Add heading and CTA inside.

### 2. Feature Highlight
Use **Shape 1** at 400px, solid color, **Morph 1** animation. Add icon and short text inside.

### 3. Image Container
Use **Shape 5**, add background image, enable overlay at 30% for text readability. Add content inside.

### 4. Decorative Element
Use **Shape 2** at 300px, gradient, **Pulse** animation, no inner content. Position next to text sections.

## ‚úÖ Best Practices

**DO:**
- Use animations sparingly (1-2 per viewport)
- Match blob colors to your theme palette
- Test on mobile (reduce size if needed)
- Add inner content for interactive blobs
- Use overlays to ensure text readability on images

**DON'T:**
- Use fast animations (2s or less) - causes distraction
- Overload page with many animated blobs
- Use blob shapes for critical UI elements (buttons, forms)
- Forget to set appropriate size for container width
- Use very bright gradients without overlay for text backgrounds

## ‚ôø Accessibility

- **Content Readability**: Ensure text inside has sufficient contrast
- **Motion Sensitivity**: Consider users with motion sensitivity - use subtle animations
- **Keyboard Navigation**: Inner blocks remain keyboard accessible
- **Screen Readers**: Blob is decorative, content inside is fully accessible

## üí° Tips & Tricks

- **Layering**: Place multiple blobs at different z-indexes for depth
- **Negative Space**: Use blobs to break up text-heavy layouts
- **Color Theory**: Use complementary gradients for visual interest
- **Size Variation**: Mix large and small blobs for dynamic compositions
- **Background Blend**: Use semi-transparent blobs over page backgrounds
- **Animation Combo**: Morph + Float provides most organic movement
- **Performance**: Limit to 3-4 animated blobs per page for smooth performance
